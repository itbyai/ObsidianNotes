这份内容本质上是在讲 Databricks 认证里 **Design Applications** 这一部分的考试重点，属于一个**考点串讲 + 样题解析**的视频字幕。

## 整体内容总结

这段内容围绕 5 个核心考点展开，每个考点都配了一个典型选择题，目的不是深入技术实现，而是让你建立一种**考试判断框架**：

1. **如何设计 prompt，让模型输出指定格式**
    
2. **如何根据业务需求选择合适的模型任务**
    
3. **如何选择链式应用里的组件**
    
4. **如何把业务目标翻译成 AI pipeline 的输入和输出**
    
5. **如何为多步骤 agent 排列工具调用顺序**
    

---

## 每一部分讲了什么

### 1）Prompt 设计：要让输出格式明确

案例是一个 customer support summarizer，要求模型输出固定格式。

核心结论：

- 最好的 prompt 不是只说“总结一下”
    
- 也不是只说“返回 JSON”
    
- **最好是直接把目标格式明确写出来，甚至给出 exact JSON format**
    

这部分强调的原则是：

- prompt 要 **clear**
    
- 要 **explicit**
    
- 要 **structured**
    

这样做的价值：

- 降低 hallucination
    
- 提高输出一致性
    
- 更适合后续程序解析
    

这其实是考试里非常常见的思路：  
**越具体、越结构化、越约束格式，通常越优。**

---

### 2）模型任务选择：业务目标决定任务类型

案例是分析产品评论，识别新出现的问题。

给出的选项包括：

- 文本摘要
    
- 命名实体识别
    
- 情感分析
    
- 意图分类
    

讲解里选的是 **sentiment analysis**。

这里想传递的考试思路是：

- 当业务需求是“发现负面趋势、识别问题、判断客户满意度变化”
    
- 那么最直接对应的任务往往是 **情感分析**
    

也就是说，这部分不是考你模型原理，而是考你：  
**看到业务需求时，能不能把它映射到正确的 AI task。**

---

### 3）链式组件选择：面向 grounded answer 的典型结构

案例是：根据 Delta Lake 里的内部政策文档回答用户问题，并且答案要基于文档。

正确思路是一个典型 **RAG 流程**：

- Retriever 找相关文档
    
- Prompt 组织上下文和问题
    
- LLM 生成回答
    

核心结论：

- 如果题目要求“基于文档回答”
    
- 那么一般不是直接把问题丢给 LLM
    
- 而是要有 **retriever + prompt + LLM** 这种结构
    

这部分其实在考你：  
**什么时候需要 RAG，什么时候只是普通 prompt 调用。**

只要看到这些关键词，你就要警觉：

- based on documents
    
- grounded answer
    
- internal knowledge
    
- policy docs
    
- Delta table / document store
    

大概率就是 RAG。

---

### 4）把业务目标转成输入输出规格

案例是：财务分析师希望系统读取 earnings transcript，并输出风险和机会摘要。

正确思路是：

- **输入**：earnings transcripts
    
- **输出**：包含 risks 和 opportunities 的结构化结果，最好是 JSON 风格
    

这里的重点不是“财务知识”，而是：  
**你能不能准确识别业务系统真正处理的输入是什么，真正交付的输出是什么。**

很多错误选项会故意换成：

- 输入变成 company name
    
- 输入变成 stock price
    
- 输出变成 score  
    这些都不是题目原始需求。
    

考试技巧就是：  
**严格贴着题干，不要自己脑补。**

---

### 5）多工具 agent 的顺序设计

案例是：一个研究助手要搜索公司财务、总结结果，再通过邮件发给利益相关者。

正确顺序是：

1. Web search
    
2. Summarizer
    
3. Email tool
    

核心逻辑很简单：

- 先获取信息
    
- 再处理信息
    
- 最后执行动作
    

这一部分考的是：  
**工具顺序是否符合任务流。**

常见陷阱是故意把 email 放前面，或者先总结后搜索。  
只要按“数据从哪来 → 如何处理 → 如何输出/执行”这个顺序想，通常就不会错。

## 这份内容的核心价值

这份字幕内容并不是教你很深的 GenAI 工程实现，而是在帮你建立一个**认证考试的答题模式**：

### 它反复强调的主线其实只有几个：

- **Prompt 要具体、格式要明确**
    
- **任务选择要贴合业务目标**
    
- **基于文档回答通常意味着 RAG**
    
- **输入输出定义必须忠于题干**
    
- **Agent 工具顺序要符合真实工作流**
