在 React 中，受控组件和非受控组件是处理表单输入的两种方式。它们之间的主要区别在于输入值的管理方式。下面详细介绍这两种组件，包括主要作用、用法和示例。

### 受控组件 (Controlled Components)

**定义**：受控组件是指表单元素的值由 React 的状态管理控制。输入的值通过 React 的状态更新，并通过事件处理函数进行管理。

#### 主要作用

1. **状态同步**：输入的值和组件的状态始终保持同步，确保组件状态的可控性。
2. **表单验证**：可以轻松地在每次输入变化时进行验证。
3. **数据处理**：可以方便地处理和提交表单数据，因为输入的值已存储在组件的状态中。

#### 用法

在受控组件中，使用 `value` 属性将输入框的值与组件的状态绑定，并通过事件处理函数更新状态。

#### 示例

以下是一个受控组件的示例：

```jsx
import React, { useState } from 'react';

const ControlledInput = () => {
  const [inputValue, setInputValue] = useState('');

  const handleChange = (event) => {
    setInputValue(event.target.value); // 更新状态
  };

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Submitted value: ${inputValue}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Controlled Input:
        <input
          type="text"
          value={inputValue} // 受控组件的值由状态控制
          onChange={handleChange} // 处理变化
        />
      </label>
      <button type="submit">Submit</button>
    </form>
  );
};

export default ControlledInput;
```

### 非受控组件 (Uncontrolled Components)

**定义**：非受控组件是指表单元素的值不由 React 的状态管理控制，而是直接从 DOM 元素中读取。使用 `ref` 来获取 DOM 元素的值。

#### 主要作用

1. **简化状态管理**：对于简单表单或不需要频繁验证的场景，非受控组件可以减少状态管理的复杂性。
2. **直接与 DOM 交互**：在需要与现有的非 React 代码（如 jQuery）集成时，非受控组件可以更方便。

#### 用法

在非受控组件中，使用 `ref` 来访问输入框的值，而不是将其存储在组件的状态中。

#### 示例

以下是一个非受控组件的示例：

```jsx
import React, { useRef } from 'react';

const UncontrolledInput = () => {
  const inputRef = useRef(null); // 创建 ref

  const handleSubmit = (event) => {
    event.preventDefault();
    alert(`Submitted value: ${inputRef.current.value}`); // 直接从 ref 获取值
  };

  return (
    <form onSubmit={handleSubmit}>
      <label>
        Uncontrolled Input:
        <input
          type="text"
          ref={inputRef} // 将 ref 绑定到输入框
        />
      </label>
      <button type="submit">Submit</button>
    </form>
  );
};

export default UncontrolledInput;
```

### 受控组件与非受控组件的比较

| 特点                  | 受控组件                               | 非受控组件                            |
|---------------------|------------------------------------|----------------------------------|
| 值管理              | 由 React 状态管理                     | 直接从 DOM 元素读取                 |
| 使用方式            | 使用 `value` 和 `onChange` 绑定状态    | 使用 `ref` 获取输入值                |
| 适用场景            | 需要实时更新、验证和控制的表单           | 简单表单或与非 React 代码集成的场景    |
| 性能                | 每次输入变化都会触发重新渲染             | 仅在表单提交时处理，性能稍好          |

### 小结

受控组件和非受控组件是 React 中处理表单输入的两种方式。选择使用哪种方式取决于具体需求。受控组件提供了更好的状态管理和验证功能，适合复杂表单；而非受控组件则简化了状态管理，更适合简单表单或与其他代码集成的场景。了解这两种组件的特点和用法，可以帮助开发者在开发过程中做出更合适的选择。
