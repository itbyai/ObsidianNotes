CSS 定位（Positioning）是一种用于控制元素在网页中的位置的技术。它允许你精确地定位和布局 HTML 元素，无论是相对于文档的其他部分，还是相对于浏览器窗口本身。

在 CSS 中，有四种常见的定位方式：

1. **Static（静态定位）**：这是默认的定位方式。静态定位的元素按照它们在HTML文档中的正常流顺序进行布局，不会受到任何定位属性的影响。
>静态定位（Static positioning）是 CSS 中最基本的定位方式之一。当元素采用静态定位时，它会按照文档流中的正常顺序进行布局，元素不会受到任何定位属性的影响，即使你对其使用了定位属性也没有效果。

>静态定位是元素的默认定位方式，即使你没有显式地指定任何定位属性，元素仍然会采用静态定位。

>特点：

>1. **默认定位方式**：静态定位是元素的默认定位方式，当你没有显式地指定其他定位方式时，元素会自动采用静态定位。

>2. **按照文档流进行布局**：静态定位的元素会按照它们在 HTML 文档中的正常流顺序进行布局。它们会在页面中占据它们在 HTML 结构中的位置，并且不受其他定位属性的影响。

>3. **不能设置偏移量**：静态定位的元素不能通过设置 `top`, `right`, `bottom`, `left` 等属性来改变它们的位置。即使你对其使用了这些属性，也不会产生任何效果。

>示例：

```css
.static-element {
    position: static;
}
```

>在这个示例中，`.static-element` 元素采用了静态定位，即使没有显式指定 `position` 属性，它也会自动采用静态定位。这个元素将按照文档流中的正常顺序进行布局，不会受到任何定位属性的影响。

3. **Relative（相对定位）**：相对定位的元素相对于其`正常位置`进行移动。你可以通过设置 `top`, `right`, `bottom`, `left` 等属性来指定元素`相对于其原始位置`的偏移量。

4. **Absolute（绝对定位）**：绝对定位的元素相对于其最近的`已定位祖先元素`进行定位。如果没有已定位的祖先元素，则相对于`最初包含它的元素`（通常是 `<body>`）进行定位。绝对定位的元素会脱离文档的正常流，并且不占据文档的布局空间。

	CSS 中的绝对定位（Absolute positioning）是一种定位元素的方式，它使得元素脱离了文档流，相对于最近的已定位祖先元素进行定位，如果没有已定位的祖先元素，则相对于最初包含它的元素进行定位（通常是 `<body>`）。绝对定位的元素不会影响其他元素的布局。
	
	使用绝对定位时，你可以通过设置 `top`, `right`, `bottom`, `left` 等属性来指定元素相对于其定位父元素（或页面的边界）的偏移量。这些属性值可以是正数、负数或百分比，表示元素相对于定位父元素的边界的距离。
	
	绝对定位的特点包括：
	
	1. **脱离文档流**：绝对定位的元素脱离了文档流，不会影响其他元素的布局，其他元素会忽略该元素的位置。
	
	2. **相对于已定位的祖先元素进行定位**：绝对定位的元素相对于其最近的已定位祖先元素进行定位。如果没有已定位的祖先元素，则相对于最初包含它的元素进行定位。
	
	3. **使用偏移量进行定位**：你可以通过设置 `top`, `right`, `bottom`, `left` 等属性来指定元素相对于定位父元素的偏移量。
	
	使用绝对定位，你可以创建各种复杂的布局和效果，如悬浮菜单、弹出窗口、模态框等。然而，需要注意的是，使用绝对定位时要谨慎，确保你的布局在不同设备和屏幕尺寸下仍然能够正常显示和工作。

6. **Fixed（固定定位）**：固定定位的元素相对于`浏览器窗口`进行定位。它们会随着页面的滚动而固定在指定的位置，不受滚动影响。

使用 CSS 定位，你可以创建各种复杂的布局和效果，如悬浮菜单、弹出窗口、滚动导航等。然而，需要注意的是，使用定位时要谨慎，确保你的布局在不同设备和屏幕尺寸下仍然能够正常显示和工作。


# 一个特殊的z-index
`z-index` 是 CSS 属性之一，用于控制定位元素的堆叠顺序（stacking order）。它定义了一个元素在堆叠上下文中的层级关系，即确定了元素在 z 轴方向上的显示顺序，其中 z 轴垂直于屏幕。

当页面上有多个定位元素重叠时，`z-index` 属性可以帮助我们决定哪个元素位于另一个元素的上方。

`z-index` 属性接受一个整数值或 `auto` 作为参数，整数值越大，元素在堆叠顺序中就越靠前，即越靠近用户。默认情况下，元素的 `z-index` 值为 `auto`，即不设置特定的层级关系。

值得注意的是，`z-index` 只对定位元素（positioned elements）有效，即设置了 `position: relative`, `position: absolute`, `position: fixed` 的元素。对于没有设置定位的元素，`z-index` 不会生效。

当元素的 `z-index` 值相同时，HTML 结构中后面出现的元素会覆盖前面出现的元素。如果想要改变元素的层级关系，可以通过设置不同的 `z-index` 值来实现。

示例：

```css
.element1 {
    position: relative;
    z-index: 1;
}

.element2 {
    position: relative;
    z-index: 2;
}
```

在这个示例中，`.element2` 元素会位于 `.element1` 元素的上方，因为它的 `z-index` 值比较大。
