在 React 中，一个组件的**执行顺序**（即生命周期）遵循一套明确的流程，取决于是函数组件还是 class 组件。这里我们重点讲函数组件的执行顺序，尤其是配合 Hooks 的现代写法。

---

## ✅ 函数组件执行顺序（首次渲染）

假设我们有如下组件：

```jsx
function MyComponent() {
  console.log("1️⃣ render start");

  const [count, setCount] = useState(0); // 2️⃣ useState 执行
  useEffect(() => {
    console.log("5️⃣ useEffect executed");
  }, []);

  console.log("3️⃣ render end");

  return (
    <div>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  );
}
```

### 执行顺序如下：

|顺序|阶段|说明|
|---|---|---|
|1️⃣|执行组件函数|每次渲染都会执行整个函数（但状态是保持的）|
|2️⃣|执行 Hook（如 `useState`）|React 根据调用顺序匹配保存的 hook 状态|
|3️⃣|返回 JSX，渲染 UI|生成虚拟 DOM，准备挂载到真实 DOM|
|4️⃣|commit 阶段|React 提交真实 DOM 更新|
|5️⃣|执行 `useEffect`|所有 DOM 渲染完后再异步执行（副作用）|

---

## ✅ 更新时的执行顺序（点击按钮后）

点击 `<button>` 会触发 `setCount`，引起组件重新 render。

重新执行的顺序如下：

1. 清除旧的 `useEffect`（如果有清理函数）
    
2. 执行组件函数（再次执行 `useState`、重新返回 JSX）
    
3. 提交更新到真实 DOM（如按钮数字更新）
    
4. 执行新的 `useEffect`

## ✅ 多个组件嵌套的执行顺序

```jsx
function App() {
  console.log("App render");
  return <Child />;
}

function Child() {
  console.log("Child render");
  return <div>Hello</div>;
}
```

输出顺序为：

```
App render
Child render
```

因为 React 是自上而下执行组件树的函数（从父到子）。

---

## ✅ Hook 的执行顺序说明

- React 内部通过“顺序”记录多个 Hook：
    
    ```js
    const [a, setA] = useState(); // Slot 0
    const [b, setB] = useState(); // Slot 1
    const ref = useRef();         // Slot 2
    ```
    
- 你必须保证每次 render 时，Hook 的调用顺序和数量是固定的。
    

---

## 📌 小结：React 函数组件执行顺序（现代版）

|阶段|行为|
|---|---|
|函数组件执行|执行函数体，读取 state、绑定事件等|
|Hook 按顺序执行|React 维护 hook index，对应内部状态|
|返回 JSX|构建虚拟 DOM|
|commit 阶段|挂载到真实 DOM|
|`useEffect` 异步执行|类似 `componentDidMount`，异步触发副作用|
