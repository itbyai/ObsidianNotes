# 10天快速突破架构师能力学习计划

## 目标定位
- 能够理解并设计 **云架构**（AWS、Azure）和 **系统架构**（微服务、分布式系统）
- 理解常用 **设计模式** 和 **架构决策**
- 能够快速 **分析业务需求并输出架构方案**
- 能够理解性能、可扩展性、安全性、可维护性等核心架构非功能需求

## 学习方法
1. **高效刷框架知识**
   - 不求全，重点掌握 **核心服务、模式、设计原则**
2. **案例驱动**
   - 以现有系统或典型架构为例，分析设计、权衡
3. **动手实战**
   - 画架构图、写架构文档、做小型原型
4. **反思总结**
   - 每天输出至少一篇架构思考或图

## 学习安排（10天）

### Day 1-2：架构思维与原则
- **内容**
  - 软件架构基础：分层架构、微服务、SOA
  - 架构设计原则：SOLID、DRY、KISS
  - 关键非功能需求：可扩展性、可维护性、可靠性
- **方法**
  - 阅读：《Software Architecture in Practice》精读核心章节
  - 案例分析：Netflix、Uber架构图
  - 输出：总结一套架构设计原则

### Day 3-4：云架构基础（AWS/Azure）
- **内容**
  - 核心服务：计算、存储、数据库、消息队列、CDN
  - 云架构模式：三层架构、微服务架构、Serverless
  - 架构设计要点：高可用、高性能、成本优化
- **方法**
  - AWS/Azure官方架构图 + 白皮书速读
  - 输出：画一个小型云应用架构图

### Day 5-6：微服务与分布式系统
- **内容**
  - 微服务设计：服务拆分、API设计、服务注册发现
  - 分布式系统核心概念：CAP定理、消息队列、事务一致性
  - 常用模式：Saga、CQRS、Event Sourcing
- **方法**
  - 参考Spring Cloud或Azure Service Fabric示例
  - 输出：写微服务拆分方案 + 数据流图

### Day 7：数据库与存储架构
- **内容**
  - 关系型 vs 非关系型数据库
  - 数据分片、缓存、索引优化
  - 高可用存储架构设计
- **方法**
  - 分析Netflix、TikTok数据存储架构
  - 输出：绘制存储方案 + 读写优化策略

### Day 8：性能与安全架构
- **内容**
  - 性能优化：负载均衡、CDN、缓存、异步处理
  - 安全架构：认证授权、加密、审计、漏洞防护
- **方法**
  - 案例：AWS Well-Architected Framework
  - 输出：安全和性能改进建议清单

### Day 9：架构决策与文档
- **内容**
  - 架构决策记录（ADR）
  - 架构图、流程图、时序图绘制
- **方法**
  - 实战：用当天学到的知识写完整的架构设计文档
  - 输出：一个小型项目的架构方案文档

### Day 10：实战演练与总结
- **内容**
  - 模拟业务需求：电商、支付、社交平台
  - 输出架构方案并讲解设计决策
- **方法**
  - 绘制架构图、标注关键决策
  - 总结10天学习的核心要点，形成笔记体系

## 工具推荐
- **架构图绘制**：Diagrams.net、Lucidchart、C4-PlantUML
- **云资源学习**：AWS Well-Architected Tool, Azure Architecture Center
- **学习资料**：
  - 《Software Architecture in Practice》
  - 《Designing Data-Intensive Applications》
  - AWS / Azure 官方白皮书
  - Martin Fowler博客

## 额外加速技巧
- 每天 **30分钟架构图实战** > 理论全刷完
- 学完一个模块立即画图输出
- 用 **案例驱动学习** > 理论堆砌
- 在学习过程中 **记录架构决策理由**，帮助快速思维迁移
