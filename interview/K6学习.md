# k6 学习指南

## 一、k6 简介
[k6](https://k6.io/) 是一个开源的负载测试工具，用 Go 编写，使用 JavaScript 编写测试脚本。特点：
- 易于使用，脚本化编写
- 支持 CI/CD 集成
- 丰富的指标和可视化
- 可扩展（支持云端运行）

## 二、学习步骤

### 1. 安装 k6
- Mac: `brew install k6`
- Linux: 使用 `apt` 或 `yum`，参考 [安装文档](https://grafana.com/docs/k6/latest/get-started/installation/)
- Windows: 使用 `choco install k6` 或下载二进制文件

### 2. 编写第一个测试脚本
创建 `script.js`：
```javascript
import http from 'k6/http';
import { sleep } from 'k6';

export default function () {
  http.get('https://test.k6.io');
  sleep(1);
}
````

运行：

```bash
k6 run script.js
```

### 3. 理解输出结果

k6 会输出指标：

- `http_reqs`: 请求数
- `http_req_duration`: 请求耗时
- `vus`: 虚拟用户数
- `iterations`: 脚本运行迭代次数
 

### 4. 配置负载场景

可以在脚本里配置虚拟用户数和持续时间：

```javascript
export const options = {
  vus: 10,   // 10 个虚拟用户
  duration: '30s', // 持续 30 秒
};
```

### 5. 增强脚本功能

- 使用 `check` 断言响应
- 使用 `Trend`, `Counter` 自定义指标
- 使用 `setup` 和 `teardown` 钩子
- 场景配置（ramping VUs、stages）
 

### 6. 集成与可视化

- 与 [Grafana + Prometheus](https://grafana.com/docs/k6/latest/results-visualization/influxdb-+-grafana/) 集成
- k6 Cloud 托管服务
- CI/CD 集成（GitHub Actions、GitLab CI、Jenkins）
