## ✅ 四、你应该准备的项目讲解内容

> 面试官常常会问你 “你在项目中是怎么用 Selenium 的？”

建议准备如下：

- 使用 Java + Selenium + TestNG 编写了 xx 测试用例，涵盖哪些业务模块。
    
- 用 Page Object 模式组织页面类，解耦元素定位和测试逻辑。
    
- 集成 Jenkins 定时跑回归测试，每天生成报告发邮件。
    
- 在测试失败时截图并生成报告，排查用例 flakiness。
    
- 如何处理动态页面、等待加载、iframe 和 window 切换等难点。

## ✅ 五、常见面试题目（精选）

|题目|回答要点|
|---|---|
|Selenium 支持哪些浏览器？|Chrome, Firefox, Safari, Edge，解释通过什么驱动连接|
|显式等待的实现方式？|`WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));`|
|如何处理下拉菜单？|`Select select = new Select(element); select.selectByVisibleText("XXX");`|
|页面加载慢怎么办？|用 `ExpectedConditions` + 显式等待|
|什么是 POM？你项目里怎么用？|页面逻辑和测试逻辑分离，便于维护和复用|
|怎么处理不稳定的 UI 元素？|加入等待机制、retry 机制，或重构测试点|
|如何与后端接口配合验证？|可以在 UI 测试中调用 API 来初始化或校验状态|
