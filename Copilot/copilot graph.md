# C#后端架构图生成提示词指南

为了生成类似这样的C#后端架构图，您可以使用以下几种提示词模式：

## 1. 基础架构分析提示词

```
请分析我的C#后端项目结构，生成一个Mermaid架构图，包括：
- API控制器层
- 应用服务层  
- 数据访问层
- 外部服务依赖
- 数据库连接

请使用subgraph来分组不同的层次。
```

## 2. 详细的技术栈提示词

```
根据我的C# Web API项目，创建一个Mermaid图表，展示：
- Controllers文件夹中的所有控制器
- Services文件夹中的业务逻辑服务
- Repositories文件夹中的数据访问层
- 外部API客户端和依赖注入的服务
- 数据库（SQL Server/Cosmos DB等）连接
- 服务之间的依赖关系

使用graph TB方向，并用subgraph分组。
```

## 3. 领域驱动设计(DDD)风格提示词

```
分析我的C# DDD架构项目，生成Mermaid架构图，包含：
- Presentation Layer (Controllers/API)
- Application Layer (Services, DTOs)
- Domain Layer (Entities, Value Objects, Domain Services)
- Infrastructure Layer (Repositories, External Services)
- 跨层依赖关系

请使用适当的颜色和样式区分不同层次。
```

## 4. 微服务架构提示词

```
为我的C#微服务项目创建架构图，展示：
- API网关或入口点
- 各个微服务的控制器
- 服务间通信（HTTP, gRPC, Message Queue）
- 数据库分离
- 外部依赖服务
- 缓存层（Redis等）

使用Mermaid的subgraph功能清晰分组。
```

## 5. 具体到您项目的提示词

```
分析我的C#配送系统后端代码，生成Mermaid架构图，重点展示：
- DeliverySegments相关的Controller、Service、Repository
- DeliveryRule相关的业务逻辑层次
- OrderTimes服务及其依赖关系
- Cosmos DB数据访问模式
- Store Service外部集成
- 搜索服务的实现架构

请使用graph TB布局，用subgraph分组不同架构层。
```

## 💡 使用建议：

1. **先扫描项目结构**：让AI工具先分析您的文件夹结构和依赖关系
2. **指定图表类型**：明确要求使用Mermaid的`graph TB`或`flowchart TB`
3. **要求分层展示**：使用`subgraph`来分组不同的架构层
4. **包含依赖关系**：明确要求显示服务之间的调用关系
5. **指定技术栈**：提及具体使用的数据库、缓存、外部服务等

您可以根据项目的具体特点调整这些提示词，这样就能生成类似您示例中的清晰架构图了。
