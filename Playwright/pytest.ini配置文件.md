`pytest.ini` 是一个配置文件，用于为 `pytest` 测试框架定义全局设置和选项。通过这个文件，你可以控制测试运行的行为，而不需要在每次运行测试时手动添加命令行参数。`pytest.ini` 允许你配置以下内容：

1. **标记（Markers）**：
   - 你可以在 `pytest.ini` 中定义自定义标记，并为它们添加描述。这样在使用这些标记时，`pytest` 不会发出警告。
```ini
   [pytest]
   markers =
       slow: marks tests as slow (deselect with '-m "not slow"')
```

2. **测试路径（Test paths）**：
   - 你可以指定测试文件所在的默认目录，这样运行测试时，`pytest` 会在该目录中查找测试文件。
```ini
   [pytest]
   testpaths = tests
```

3. **忽略的测试文件或目录（Test ignore paths）**：
   - 你可以通过 `norecursedirs` 配置项来指定不希望 `pytest` 递归查找测试文件的目录。
```ini
   [pytest]
   norecursedirs = .git venv
```

4. **添加命令行选项**：
   - 你可以通过 `addopts` 配置项添加默认的命令行选项，这样每次运行 `pytest` 时，都会自动使用这些选项。
```ini
   [pytest]
   addopts = -v --maxfail=2 --tb=short
```

5. **Python 导入路径（Python import paths）**：
   - 你可以通过 `pythonpath` 配置项添加自定义的 Python 导入路径，使 `pytest` 在这些路径中查找模块。
```ini
   [pytest]
   pythonpath = src
```

6. **日志配置（Logging configuration）**：
   - 你可以在 `pytest.ini` 中配置日志记录的级别、格式和日志输出文件。
```ini
   [pytest]
   log_cli = true
   log_cli_level = INFO
```

7. **禁用警告（Disable warnings）**：
   - 你可以通过 `filterwarnings` 配置项禁用某些特定类型的警告。
```ini
   [pytest]
   filterwarnings =
       ignore::DeprecationWarning
```

Pytest的工作原理
通过 `pytest.ini` 文件，你可以统一管理 `pytest` 的配置，使测试环境更易于维护和管理。如果项目需要跨多个环境或团队协作，`pytest.ini` 是一个非常有用的工具。

`pytest.ini` 文件的工作原理涉及 `pytest` 在启动时如何读取和应用配置文件的过程。这里是 `pytest.ini` 的工作原理概述：

1. **配置文件的读取顺序**：
   - 当你运行 `pytest` 时，它会自动查找并读取项目根目录下的配置文件。`pytest` 支持多个配置文件格式，包括 `pytest.ini`、`tox.ini` 和 `setup.cfg`。它会按照以下优先级顺序读取这些文件：
     1. `pytest.ini`
     2. `tox.ini`
     3. `setup.cfg`

   - 如果同一目录下存在多个配置文件，`pytest` 将首先使用优先级较高的文件（例如，`pytest.ini` 的优先级高于 `tox.ini`）。

2. **解析配置文件**：
   - `pytest` 解析 `pytest.ini` 文件的内容，将其中的配置信息加载到运行时环境中。这些配置包括全局选项、命令行参数、标记、忽略目录、测试路径等。

3. **应用配置**：
   - 解析完配置文件后，`pytest` 将应用这些配置到测试运行中。具体来说：
     - **命令行选项** (`addopts`) 会作为默认参数自动添加到 `pytest` 命令中。
     - **标记** (`markers`) 被注册，使其在测试中使用时不会引发警告。
     - **测试路径** (`testpaths`) 指定了默认的测试文件或目录，`pytest` 会在这些路径中查找测试。
     - **忽略路径** (`norecursedirs`) 使 `pytest` 跳过某些目录，不会在这些目录中递归查找测试文件。
     - **日志设置** 和 **警告过滤** 等配置也会影响 `pytest` 的行为，使得日志记录和警告处理方式符合配置要求。

4. **结合命令行参数**：
   - 如果你在运行 `pytest` 时通过命令行提供了参数，这些参数会与 `pytest.ini` 中的配置结合使用。命令行参数通常会覆盖 `pytest.ini` 中的对应配置。例如，如果你在 `pytest.ini` 中配置了 `addopts = -v`，但在命令行中使用了 `-q` 选项，最终的测试运行会使用 `-q` 选项。

5. **动态配置**：
   - 某些配置选项还可以通过钩子函数或插件进行动态修改。例如，使用 `conftest.py` 文件可以动态调整某些选项的值，这些调整会在 `pytest.ini` 配置加载后进一步影响 `pytest` 的运行。

总结来说，`pytest.ini` 的工作原理是 `pytest` 在启动时自动加载并解析它，将文件中的设置应用到测试运行环境中，使得配置统一且自动化。这样可以减少手动配置的复杂性，并保证测试在不同环境下的一致性。

如果没有任何配置

如果你的项目中没有任何配置文件（如 `pytest.ini`、`tox.ini`、`setup.cfg`），那么 `pytest` 将使用其内置的默认设置。这些默认设置是 `pytest` 在没有任何用户配置时预设的行为和选项。具体来说，默认设置的获取和应用机制如下：

### 1. **内置默认设置**：
   - `pytest` 自带一套默认设置，这些设置在没有配置文件的情况下自动生效。例如：
     - **测试发现**：`pytest` 会自动查找符合命名约定的测试文件和函数。默认情况下，它会递归地查找文件名以 `test_` 开头或以 `_test.py` 结尾的文件，以及这些文件中以 `test_` 开头的函数。
     - **日志记录**：默认情况下，日志记录功能是开启的，但日志的详细程度和格式使用内置的默认值。
     - **命令行输出**：`pytest` 默认会在命令行中显示测试进度、测试结果和失败的详细信息，使用标准的输出格式。
     - **插件**：如果项目中安装了 `pytest` 插件，这些插件也会自动加载并使用默认设置。

### 2. **命令行参数**：
   - 如果你在运行 `pytest` 时提供了命令行参数，这些参数会直接覆盖 `pytest` 的内置默认设置。例如，如果你运行 `pytest -v`，则会启用详细输出模式，即使没有配置文件也能使用此选项。

### 3. **环境变量**：
   - `pytest` 还支持通过环境变量来影响其行为。虽然这种方式不如配置文件常见，但在某些情况下可以使用。例如，设置 `PYTEST_ADDOPTS` 环境变量可以为所有 `pytest` 命令添加默认选项。

### 4. **`conftest.py` 文件**：
   - 即使没有 `pytest.ini` 或其他配置文件，项目中的 `conftest.py` 文件仍然可以用来定义一些局部的配置和钩子函数。这些文件位于测试目录中，可以动态地影响 `pytest` 的行为，如自定义标记、钩子函数等。

### 5. **pytest 插件**：
   - `pytest` 自带一些内置插件，这些插件提供了额外的功能和默认行为。例如，`pytest` 的参数化、跳过测试、失败重试等功能都是通过插件实现的，即使没有配置文件，这些插件也会以默认设置运行。

### 6. **自定义配置**：
   - 如果你有需要，你可以在命令行直接使用 `pytest` 提供的多种选项来临时调整默认设置。例如，使用 `--maxfail=2` 来限制最大失败次数，或者使用 `--tb=short` 来改变错误跟踪的显示方式。

通过这些默认机制，`pytest` 可以在没有任何用户配置文件的情况下，仍然有效地运行测试并提供合理的默认行为。这使得它能够在任何环境下开箱即用，而用户可以根据需要逐步增加配置文件和选项来定制化其行为。
