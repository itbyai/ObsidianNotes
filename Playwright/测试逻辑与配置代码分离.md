将测试逻辑与配置代码分离是编写可维护、可扩展测试的重要实践。以下是如何在测试中实现这种分离的最佳实践：

### 1. 使用 `conftest.py` 进行配置

在 `conftest.py` 文件中定义所有与测试环境和配置相关的夹具和配置。这包括浏览器实例、页面对象、API 客户端等。这些配置将会在测试执行前被准备好，并在测试完成后进行清理。

```python
# conftest.py
import pytest
from playwright.sync_api import sync_playwright

@pytest.fixture(scope="module")
def browser():
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=True)
        yield browser
        browser.close()

@pytest.fixture
def page(browser):
    page = browser.new_page()
    yield page
    page.close()

@pytest.fixture
def api_client():
    # Example API client setup
    client = SomeApiClient()
    yield client
    client.close()
```

### 2. 在测试文件中仅编写测试逻辑

测试文件应该专注于测试逻辑和断言，不包含与测试环境和配置相关的代码。使用 `conftest.py` 中定义的夹具来获取必要的资源和服务。

```python
# test_example.py
def test_page_title(page):
    page.goto('https://example.com')
    assert page.title() == 'Example Domain'

def test_element_exists(page):
    page.goto('https://example.com')
    assert page.locator('h1').inner_text() == 'Example Domain'

def test_api_response(api_client):
    response = api_client.get('/some-endpoint')
    assert response.status_code == 200
    assert response.json() == {"expected": "value"}
```

### 3. 使用配置文件来管理环境特定的设置

可以通过配置文件（如 `.env` 文件或 JSON/YAML 配置文件）来管理环境特定的设置。例如，可以在 `conftest.py` 中读取这些配置文件以获取环境变量或其他配置。

```python
# conftest.py
import pytest
import os
from dotenv import load_dotenv
from playwright.sync_api import sync_playwright

load_dotenv()  # Load environment variables from a .env file

@pytest.fixture(scope="module")
def browser():
    headless = os.getenv("HEADLESS", "true") == "true"
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=headless)
        yield browser
        browser.close()
```

### 4. 使用工厂模式和夹具生成

对于复杂的测试环境，可以使用工厂模式或夹具生成器来创建各种配置或对象。这可以帮助在不同的测试用例中生成不同的配置或状态。

```python
# conftest.py
import pytest
from playwright.sync_api import sync_playwright

@pytest.fixture
def custom_browser(request):
    headless = request.config.getoption("--headless")
    with sync_playwright() as p:
        browser = p.chromium.launch(headless=headless)
        yield browser
        browser.close()

def pytest_addoption(parser):
    parser.addoption("--headless", action="store_true", default=False, help="Run tests in headless mode")
```

### 5. 将配置和逻辑分离到模块化的组件

对于更复杂的项目，可以将配置和逻辑分离到模块化的组件中，并在 `conftest.py` 中导入这些组件。例如，将 API 客户端或数据生成器放在单独的模块中，并在测试中使用这些模块。

```python
# api_client.py
class ApiClient:
    def __init__(self):
        # Setup client
        pass

    def get(self, endpoint):
        # Make a GET request
        pass

    def close(self):
        # Clean up client
        pass

# conftest.py
import pytest
from api_client import ApiClient

@pytest.fixture
def api_client():
    client = ApiClient()
    yield client
    client.close()
```

### 总结

1. **使用 `conftest.py`**: 将配置和夹具代码集中在 `conftest.py` 中，确保测试文件只包含测试逻辑。
2. **测试文件**: 仅包含测试逻辑和断言，使用夹具获取资源。
3. **配置文件**: 使用配置文件管理环境特定设置，保持代码清晰和灵活。
4. **模块化组件**: 将复杂的配置和逻辑分离到独立模块中，保持代码的可维护性。

这种分离的设计使得测试代码更具可读性、可维护性和可扩展性。
