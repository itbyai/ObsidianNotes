当选择GitHub作为Repo Hosting Service时，DevOps专业人员可能需要掌握以下知识点：

1. **Git基础：**
   - 理解Git的基本概念，包括版本控制、分支、提交、合并等。
   - 熟悉基本的Git命令，如`git clone`、`git commit`、`git push`、`git pull`等。

2. **GitHub账户和仓库管理：**
   - 创建GitHub账户并了解GitHub的基本用户界面。
   - 创建和管理GitHub仓库，包括公共和私有仓库。
   - 设置仓库的基本信息、访问权限和协作功能。

3. **分支管理：**
   - 了解如何创建、合并和删除分支。
   - 掌握分支的最佳实践，如Feature分支、Release分支、Hotfix分支等。

4. **Pull Requests和Code Review：**
   - 了解Pull Request的概念和作用。
   - 学会创建和审查Pull Requests，进行代码评审。
   - 使用GitHub的评论和反馈功能，促进团队协作。

5. **GitHub Actions：**
   - 掌握GitHub Actions的基本概念，了解它是如何支持持续集成和持续部署的。
   - 创建和配置GitHub Actions workflows，执行自动化任务。

6. **Issue和项目管理：**
   - 使用GitHub Issues来跟踪和管理工作任务。
   - 创建和管理GitHub项目，组织任务和问题。

7. **Markdown语法：**
   - GitHub使用Markdown语法用于文档编写、README文件等。
   - 熟悉Markdown的基本语法，包括标题、列表、链接等。

8. **GitHub API：**
   - 了解GitHub的API，以便在需要时通过编程方式与GitHub进行交互。
   - 使用GitHub REST API或GraphQL API执行各种操作。

9. **安全性和权限管理：**
   - 理解GitHub的安全特性，如两步验证、Access Token等。
   - 管理仓库和组织的访问权限，确保代码和信息的安全性。

10. **GitHub CLI：**
    - 了解GitHub命令行工具（GitHub CLI），它提供了在终端中管理GitHub仓库的便捷方式。

11. **团队协作：**
    - 学会与团队成员协作，使用GitHub的协作功能，如团队讨论、合作审查等。

通过掌握以上知识点，DevOps专业人员能够更有效地在GitHub上进行版本控制、协作和自动化。 GitHub作为一个强大的Repo Hosting Service，提供了丰富的功能来支持团队在DevOps流程中的各个阶段。
