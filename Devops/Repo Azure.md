当选择Azure DevOps Services（Azure Repos）作为Repo Hosting Service时，DevOps专业人员可能需要掌握以下知识点：

1. **Azure DevOps基础：**
   - 熟悉Azure DevOps Services平台，了解Azure Repos的基本功能。
   - 创建和管理Azure DevOps账户，设置组织和项目。

2. **Git基础：**
   - 具备Git的基本概念，包括版本控制、分支、提交、合并等。
   - 熟悉基本的Git命令，如`git clone`、`git commit`、`git push`、`git pull`等。

3. **Azure Repos仓库管理：**
   - 创建和管理Azure Repos仓库，包括Git仓库和TFVC（Team Foundation Version Control）仓库。
   - 设置仓库的访问权限和分支策略。

4. **分支策略和合并请求：**
   - 制定适合团队的分支管理策略。
   - 使用Azure Repos的Pull Requests来进行代码审查和合并工作。

5. **CI/CD集成：**
   - 了解Azure Pipelines的基本概念，配置CI/CD pipelines。
   - 编写Azure Pipelines配置文件，定义构建、测试和部署流程。

6. **Azure Artifacts：**
   - 了解Azure Artifacts，用于存储和管理软件包（NuGet、npm、Maven等）。
   - 集成Azure Artifacts到CI/CD流程，确保依赖的一致性和可用性。

7. **Azure DevOps Extensions和API：**
   - 了解Azure DevOps Marketplace中的扩展和插件，可用于增强Azure DevOps功能。
   - 学习如何使用Azure DevOps REST API进行自动化操作。

8. **环境和变量管理：**
   - 使用Azure Pipelines中的环境变量和参数，管理构建和部署的环境。
   - 利用Azure Pipelines的多阶段（stages）功能，实现多环境部署。

9. **安全性和权限管理：**
   - 理解Azure DevOps的安全特性，包括访问控制、项目权限等。
   - 管理仓库和项目的访问权限，确保代码和信息的安全性。

10. **测试和质量管理：**
    - 使用Azure Test Plans进行测试计划和执行。
    - 集成测试结果到CI/CD流程，实现自动化测试。

11. **持续监控和报告：**
    - 利用Azure DevOps的监控和报告功能，监视CI/CD执行、代码质量等指标。
    - 利用Azure Boards的项目管理功能，进行工作项追踪和报告。

12. **Azure DevOps CLI：**
    - 了解Azure DevOps命令行工具（Azure DevOps CLI），提供了在终端中管理Azure DevOps资源的便捷方式。

通过掌握这些知识点，DevOps专业人员能够更好地利用Azure DevOps Services进行版本控制、持续集成和持续部署，实现更高效的协作和交付流程。 Azure DevOps Services作为一体化的DevOps平台，提供了全方位的支持，适用于整个DevOps生命周期。
