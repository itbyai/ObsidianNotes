## 🧩 一、技术面试题（Technical Interview Questions）

这些题一般来自 **自动化测试框架、API 测试、CI/CD、分布式系统** 四大模块。

### 🎯 1. 自动化框架与脚本设计

1. 请解释你在项目中如何将手工回归测试转化为自动化测试？
2. 你在 Playwright 中如何处理动态元素？
3. Playwright 与 Selenium 的主要区别是什么？在银行项目中你会更推荐哪个？为什么？
4. 你如何在大型项目中管理测试数据和环境依赖？
5. 如果测试脚本偶尔失败但手动执行成功，你会如何排查？
6. 你如何设计一个可扩展的 Page Object Model 框架？
7. 你如何处理跨浏览器兼容测试？

---

### 🌐 2. API / 后端测试（重点考核）

1. 使用 RestAssured 编写 API 测试时，如何验证响应中的 JSON schema？
    
2. 什么是契约测试（Contract Testing）？为什么它在微服务架构中重要？
    
3. 如何在 Postman 或 RestAssured 中进行身份验证（Authentication）？
    
4. 请举例说明如何在测试中验证幂等性（Idempotency）。
    
5. 你如何测试微服务间的依赖或调用链？
    
6. 如果一个 API 响应延迟很高，你会如何确定是后端问题还是测试环境问题？
    

---

### ⚙️ 3. CI/CD 与 DevOps 集成

1. 你如何在 Jenkins 或 Azure DevOps Pipeline 中集成自动化测试？
    
2. 如何设置 pipeline 在构建失败时自动触发测试报告？
    
3. 你是否使用过 Docker 或容器化测试环境？如何维护一致性？
    
4. 请说明你在版本控制中如何管理测试代码分支（Git workflow）。
    
5. 你如何确保测试结果能即时反馈给开发或BA？

## 💬 二、行为面试题（Behavioural Interview Questions）

## 🏦 三、银行类岗位与一般技术公司面试的区别

|方面|银行类职位（如 Westpac）|普通科技公司|
|---|---|---|
|**安全与合规**|非常重视。测试人员需理解数据隐私、监管要求（如 PCI-DSS、APRA）|相对宽松，重点在交付速度|
|**测试范围**|更关注系统集成、回归与稳定性|偏向创新与快速迭代测试|
|**质量文化**|“零错误上线”文化，容错率极低|强调快速修复和持续发布|
|**面试风格**|行为题比例高（40-50%），需体现风险控制意识|技术深度更高，但流程问题少|
|**软技能要求**|沟通协调、stakeholder management|偏重个人贡献与技术独立性|
|**工具侧重点**|Azure DevOps、Jenkins、RestAssured、Playwright|GitHub Actions、Cypress、Postman、Docker 等|
