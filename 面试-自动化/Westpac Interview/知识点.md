### ✅ Selenium + Java 简短问答合集

1. **What is Selenium?**
    
    > Selenium is an open-source framework for automating web applications. It supports multiple browsers and languages.
    
2. **What is WebDriver?**
    
    > WebDriver is an interface used to control browsers programmatically. Each browser has its own driver (e.g., ChromeDriver).
    
3. **How do you locate elements?**
    
    > By ID, name, className, xpath, cssSelector, linkText, etc.
    
4. **What’s the difference between Implicit and Explicit Wait?**
    
    > Implicit wait is global, applies to all elements. Explicit wait targets specific conditions.
    
5. **How do you handle dropdowns?**
    
    > Use `Select` class:  
    > `Select select = new Select(element); select.selectByVisibleText("Option");`
    
6. **How do you handle alerts/popups?**
    
    > `driver.switchTo().alert().accept();`
    
7. **How do you handle frames or iframes?**
    
    > `driver.switchTo().frame("frameName");`
    
8. **How do you switch between windows/tabs?**
    
    > `driver.getWindowHandles()` and `driver.switchTo().window(handle);`
    
9. **What is Page Object Model (POM)?**
    
    > A design pattern that separates page locators and test logic into different classes for better maintainability.
    
10. **How do you run tests using TestNG?**
    

> With `@Test`, `@BeforeMethod`, `@AfterMethod` annotations. XML config files define suite structure.

11. **How do you take a screenshot on failure?**
    

> Use `TakesScreenshot`:  
> `((TakesScreenshot) driver).getScreenshotAs(OutputType.FILE);`

12. **How do you use Java with Selenium?**
    

> Java is used to write test logic, manage objects, and integrate with frameworks like TestNG, Maven, and Jenkins.

13. **How do you handle dynamic elements?**
    

> Use explicit waits + stable locators (like relative XPath or CSS).

14. **How do you integrate with CI tools like Jenkins?**
    

> Use Maven/TestNG, trigger test execution in Jenkins pipeline or freestyle jobs.

15. **What is your test automation framework structure?**
    

> Follows POM, has base classes, utilities, page classes, and test cases. Organized for reusability and maintainability.
