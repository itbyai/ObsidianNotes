在 React 中，**Reacting to Events（响应事件）** 是组件交互的核心机制之一。它允许你捕捉用户操作（如点击、输入、悬停等），并根据这些操作更新状态、触发逻辑。

---

## 🧠 一句话总结：

> React 响应事件的方式类似于原生 HTML，但采用了 **驼峰命名法（camelCase）**，并使用 **JSX 语法绑定事件处理函数**。

---

## ✅ 基础语法（事件绑定）

```jsx
<button onClick={handleClick}>Click me</button>
```

- `onClick` 是 React 的事件属性，对应原生的 `onclick`
    
- `handleClick` 是你定义的函数（事件处理器）
    

---

## 🎯 常用的事件类型（React 中的常用事件）

|事件类型|React 属性|示例|
|---|---|---|
|鼠标事件|`onClick`|`<button onClick={...} />`|
||`onDoubleClick`||
||`onMouseEnter`||
||`onMouseLeave`||

|键盘事件|`onKeyDown`|`<input onKeyDown={...} />`|

|媒体事件|`onPlay`, `onPause`, `onEnded` 等||

## 🛠 如何定义事件处理函数（Handler）

可以使用普通函数或箭头函数：

```jsx
function handleClick() {
  alert('You clicked!');
}

// 或者
const handleClick = () => {
  alert('Clicked!');
}
```

并绑定到组件中：

```jsx
<button onClick={handleClick}>Click Me</button>
```

---

## 🌀 事件对象（SyntheticEvent）

React 使用一个叫做 **SyntheticEvent** 的合成事件系统，它对原生 DOM 事件做了封装，具有一致的行为：

```jsx
function handleClick(event) {
  console.log(event.target); // 获取被点击的元素
}
```

可以使用 `event.preventDefault()` 和 `event.stopPropagation()` 等常见方法。

---

## 📌 注意事项：

1. **事件名必须使用 camelCase**，如 `onClick`，不能写成 `onclick`
    
2. **事件值必须是函数**，不能写字符串，如不能写成 `<button onClick="doSomething()">`
    
3. React 会在组件卸载时自动清理事件监听器，不需要手动解绑（除非在 `useEffect` 中添加全局监听）
    

---

## 🧩 高级：事件处理与 state、props 结合使用

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  const handleAdd = () => setCount(count + 1);

  return <button onClick={handleAdd}>Count: {count}</button>;
}
```

---

## 🔒 安全提示

- 避免在 JSX 中内联定义复杂逻辑的事件处理器
    
- 如果涉及全局事件监听（如 `window.addEventListener`），要在组件卸载时移除
