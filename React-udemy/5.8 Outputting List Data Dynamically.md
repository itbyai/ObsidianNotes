# React 项目里“动态输出列表数据”（Outputting List Data Dynamically）详解

下面按从易到难的顺序，带你把列表从“能渲染”做到“好维护、好扩展、性能稳”。

---

## 1) 基础：用 `Array.map` 渲染列表

```jsx
export default function TodoList() {
  const todos = [
    { id: 1, text: "读文档" },
    { id: 2, text: "写代码" },
  ];
  return (
    <ul>
      {todos.map(item => (
        <li key={item.id}>{item.text}</li>
      ))}
    </ul>
  );
}
```

要点：**一定给每个直接返回的元素加 `key`**，用于让 React 识别“同一个列表项”的身份。

---

## 2) `key` 的 5 条硬规则

1. **稳定且唯一**：优先使用后端提供的稳定 ID。
    
2. **不要用数组下标当 key**（会导致重排后状态错位）。只在列表完全静态、不会增删重排时才考虑。
    
3. **key 在兄弟节点范围唯一即可**，不必全局唯一。
    
4. **把 key 放在 map 的直接返回元素上**，不是放到子元素里。
    
5. **动态切分/分组时保证 key 不变**：例如 `key={`${product.id}-card`}`。
    

---

## 3) 条件渲染、占位与空状态

```jsx
function ProductList({ products }) {
  if (!products) return <p>Loading…</p>;
  if (products.length === 0) return <p>暂无数据</p>;

  return (
    <ul>
      {products.map(p => (
        <li key={p.id}>
          {p.name} {p.inStock ? null : <em>(缺货)</em>}
        </li>
      ))}
    </ul>
  );
}
```

- 习惯写清 **loading / empty / error** 分支，交互更友好。
    

---

## 4) 事件与交互（在列表中绑定事件）

```jsx
export default function SelectableList() {
  const [selectedId, setSelectedId] = useState(null);
  const items = [{ id: 1, name: "A" }, { id: 2, name: "B" }];

  return (
    <ul>
      {items.map(item => (
        <li key={item.id}>
          <button
            onClick={() => setSelectedId(item.id)}
            className={selectedId === item.id ? "active" : ""}
          >
            选择 {item.name}
          </button>
        </li>
      ))}
    </ul>
  );
}
```

- **避免把数组下标传给事件**，用 `item.id` 更稳。
    

---

## 5) 列表项组件化（拆分成可复用的 ListItem）

```jsx
function TodoItem({ todo, onToggle }) {
  return (
    <li>
      <label>
        <input
          type="checkbox"
          checked={todo.done}
          onChange={() => onToggle(todo.id)}
        />
        {todo.text}
      </label>
    </li>
  );
}

export default function TodoList() {
  const [todos, setTodos] = useState([
    { id: 1, text: "读文档", done: false },
    { id: 2, text: "写代码", done: true },
  ]);

  const onToggle = (id) =>
    setTodos(prev =>
      prev.map(t => t.id === id ? { ...t, done: !t.done } : t)
    );

  return (
    <ul>
      {todos.map(t => (
        <TodoItem key={t.id} todo={t} onToggle={onToggle} />
      ))}
    </ul>
  );
}
```

- 组件化后，逻辑清晰，测试友好。

## 6) TypeScript 类型安全（更稳的 props 与列表）

```tsx
type Todo = { id: number; text: string; done: boolean };

function TodoItem({ todo, onToggle }: {
  todo: Todo;
  onToggle: (id: number) => void;
}) {
  return (
    <li>
      <input
        type="checkbox"
        checked={todo.done}
        onChange={() => onToggle(todo.id)}
      />
      {todo.text}
    </li>
  );
}
```

- 定义 `Todo` 类型让 IDE/编译期帮你兜底。

## 7) 异步数据：获取、状态与健壮性

```jsx
function Users() {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let active = true;
    (async () => {
      try {
        const res = await fetch("/api/users");
        if (!res.ok) throw new Error("网络错误");
        const json = await res.json();
        if (active) setData(json);
      } catch (e) {
        if (active) setError(e);
      } finally {
        if (active) setLoading(false);
      }
    })();
    return () => { active = false; };
  }, []);

  if (loading) return <p>Loading…</p>;
  if (error) return <p>出错了：{String(error)}</p>;
  if (!data?.length) return <p>暂无用户</p>;

  return (
    <ul>
      {data.map(u => <li key={u.id}>{u.name}</li>)}
    </ul>
  );
}
```

- `active` 标志避免**卸载后 setState** 的警告。
    
- 服务端渲染/并发特性下，也可考虑 `Suspense` + 数据库库。

## 8) 性能与大列表

- **`React.memo`**：列表项纯展示时包一层，减少重复渲染。
    
- **`useCallback / useMemo`**：稳定回调与派生列表，避免子组件重复计算。
    
- **虚拟化（列表窗口化）**：上万条时用 `react-window`/`react-virtualized`，只渲染可见区域.
    

```jsx
const Row = React.memo(({ data, index, style }) => (
  <div style={style}>{data[index].title}</div>
));
```

> 大多数中小列表不必过早优化；先正确、再优化。

---

## 9) 过滤、排序、搜索（派生数据）

```jsx
function ProductList({ products, query, onlyInStock }) {
  const view = useMemo(() => {
    let list = products;
    if (query) list = list.filter(p => p.name.toLowerCase().includes(query.toLowerCase()));
    if (onlyInStock) list = list.filter(p => p.inStock);
    return list.sort((a, b) => a.name.localeCompare(b.name));
  }, [products, query, onlyInStock]);

  return (
    <ul>
      {view.map(p => <li key={p.id}>{p.name}</li>)}
    </ul>
  );
}
```

- 把“视图用的列表”用 `useMemo` 派生，避免每次 render 都做重计算。

## 10) 嵌套列表与 `Fragment` 的 `key`

```jsx
return products.map(cat => (
  <React.Fragment key={cat.id}>
    <h3>{cat.name}</h3>
    <ul>
      {cat.items.map(item => <li key={item.id}>{item.label}</li>)}
    </ul>
  </React.Fragment>
));
```

- 需要一个“包裹但不多渲染的容器”时，`<></>` 或 `Fragment`，**同样要写 key**。

## 12) 常见坑与清单（Checklist）

-  **key 稳定**：不要用 index；确保增删重排后状态不串。
    
-  **空/错/载入状态**：避免空白屏。
    
-  **事件用 ID**：不要把数组下标当身份。
    
-  **提取 ListItem**：逻辑更聚焦、便于复用与测试。
    
-  **派生列表用 `useMemo`**：过滤/排序/搜索更高效。
    
-  **大列表虚拟化**：成百上千条时再上。
    
-  **TypeScript**：给数据与回调加上类型。
    
-  **样式范围**：考虑 CSS Modules 或 CSS-in-JS 防止冲突。

### 一段完整的小例子（含搜索 + 选择 + 样式）

```jsx
import { useMemo, useState } from "react";
import "./styles.css";

const DATA = [
  { id: "a1", name: "Apple", inStock: true },
  { id: "b2", name: "Banana", inStock: false },
  { id: "c3", name: "Cherry", inStock: true },
];

function Row({ item, active, onSelect }) {
  return (
    <li className={active ? "row active" : "row"} onClick={() => onSelect(item.id)}>
      {item.name} {!item.inStock && <em>(缺货)</em>}
    </li>
  );
}

export default function App() {
  const [q, setQ] = useState("");
  const [selected, setSelected] = useState(null);

  const view = useMemo(() => {
    const k = q.trim().toLowerCase();
    return k ? DATA.filter(d => d.name.toLowerCase().includes(k)) : DATA;
  }, [q]);

  return (
    <div>
      <input placeholder="搜索…" value={q} onChange={e => setQ(e.target.value)} />
      <ul>
        {view.map(item => (
          <Row
            key={item.id}
            item={item}
            active={selected === item.id}
            onSelect={setSelected}
          />
        ))}
      </ul>
    </div>
  );
}
```

```css
.row { padding: 6px 10px; cursor: pointer; }
.row.active { background: #f5f5f5; color: #d40000; font-weight: 600; }
```

---

**一句话总结**：动态列表的关键是**稳定的 key + 清晰的状态分支 + 合理的组件拆分**，在需要时再做性能优化（memo/虚拟化/派生数据）。
