React 组件的渲染主要发生在以下几个时机：

---

# **📌 1. 组件何时渲染？**

|**渲染时机**|**触发原因**|**示例**|
|---|---|---|
|**初次渲染（挂载）**|组件第一次插入 DOM|`useEffect(() => {...}, [])` 触发|
|**状态更新**|`useState` 触发重新渲染|`setCount(count + 1)`|
|**属性（Props）变化**|父组件传入的 props 发生变化|`<Child name="A" /> → <Child name="B" />`|
|**Context 变化**|`useContext` 依赖的值发生变化|`ThemeContext.Provider` 更新|
|**父组件重新渲染**|父组件自身重新渲染，子组件会随之重新渲染|`App` 组件 `useState` 变更导致子组件渲染|
|**强制渲染**|`forceUpdate()` 或 `setState({})` 触发|`useReducer` 触发无效更新|

---

# **📌 2. 详细分析 React 组件的渲染流程**

## **✅ 1. 组件初次渲染（Mounting）**

组件**第一次插入 DOM** 时会渲染：

```tsx
import { useEffect } from "react";

function MyComponent() {
  useEffect(() => {
    console.log("组件初次渲染！");
  }, []); // 空数组表示仅在初次渲染时执行

  return <div>Hello, React!</div>;
}
```

📌 `useEffect(() => {...}, [])` 在**组件首次渲染时触发**，后续不会再次触发。

---

## **✅ 2. 组件因 `useState` 更新而重新渲染**

```tsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>计数: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
```

📌 `setCount(count + 1)` 触发**组件重新渲染**，React 会用新的 `count` 值更新 UI。

---

## **✅ 3. 组件因 `props` 变化而重新渲染**

```tsx
function Child({ name }) {
  console.log("子组件渲染", name);
  return <p>子组件: {name}</p>;
}

function Parent() {
  const [name, setName] = useState("Alice");

  return (
    <div>
      <Child name={name} />
      <button onClick={() => setName("Bob")}>修改 Name</button>
    </div>
  );
}
```

📌 `setName("Bob")` 触发 `Parent` 重新渲染，导致 `Child` 也重新渲染。

---

## **✅ 4. `useContext` 变化时触发渲染**

```tsx
import { createContext, useContext, useState } from "react";

const ThemeContext = createContext("light");

function Child() {
  const theme = useContext(ThemeContext);
  console.log("子组件渲染", theme);
  return <p>当前主题: {theme}</p>;
}

function Parent() {
  const [theme, setTheme] = useState("light");

  return (
    <ThemeContext.Provider value={theme}>
      <Child />
      <button onClick={() => setTheme("dark")}>切换主题</button>
    </ThemeContext.Provider>
  );
}
```

📌 `setTheme("dark")` 更新 `ThemeContext`，导致 `Child` 组件重新渲染。

---

## **✅ 5. `forceUpdate` 触发强制渲染**

```tsx
import { useReducer } from "react";

function MyComponent() {
  const [, forceUpdate] = useReducer(x => x + 1, 0);

  console.log("组件重新渲染！");

  return <button onClick={forceUpdate}>强制渲染</button>;
}
```

📌 `useReducer(x => x + 1, 0)` 每次调用都会**强制触发组件渲染**。

---

# **📌 3. 组件渲染的优化**

有时组件会**不必要地重新渲染**，可以使用以下方法优化：

### **✅ 1. `React.memo()` 避免不必要的 `props` 变化**

```tsx
import React from "react";

const Child = React.memo(({ name }) => {
  console.log("子组件渲染", name);
  return <p>子组件: {name}</p>;
});

function Parent() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <Child name="Alice" />
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
```

📌 `React.memo()` 使 `Child` **仅在 `props.name` 变化时重新渲染**。

---

### **✅ 2. `useCallback()` 避免不必要的函数重新创建**

```tsx
import { useState, useCallback } from "react";

function Child({ onClick }) {
  console.log("子组件渲染");
  return <button onClick={onClick}>子组件按钮</button>;
}

const MemoizedChild = React.memo(Child);

function Parent() {
  const [count, setCount] = useState(0);

  const handleClick = useCallback(() => {
    console.log("子组件按钮点击");
  }, []);

  return (
    <div>
      <MemoizedChild onClick={handleClick} />
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
```

📌 `useCallback()` 避免了 `handleClick` 重新创建，`React.memo()` 让 `Child` **不会因 `Parent` 重新渲染**。

---

### **✅ 3. `useMemo()` 避免重复计算**

```tsx
import { useState, useMemo } from "react";

function App() {
  const [count, setCount] = useState(0);
  const expensiveValue = useMemo(() => {
    console.log("计算中...");
    return count * 2;
  }, [count]);

  return (
    <div>
      <p>计算结果: {expensiveValue}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
```

📌 `useMemo()` **避免 `expensiveValue` 在 `count` 不变时重复计算**。

---

# **📌 4. 总结**

|**渲染时机**|**触发方式**|
|---|---|
|**初次渲染**|组件挂载（`useEffect(() => {...}, [])`）|
|**状态变化**|`useState` 或 `useReducer` 触发|
|**Props 变化**|父组件 `props` 变更|
|**Context 变化**|`useContext` 依赖的值更新|
|**父组件重新渲染**|直接或间接影响子组件|
|**强制渲染**|`useReducer` 触发无效更新|

---

# **📌 5. 组件渲染优化方法**

|**优化方式**|**作用**|
|---|---|
|`React.memo()`|避免 `props` 不变时的渲染|
|`useCallback()`|避免函数在每次渲染时重新创建|
|`useMemo()`|避免重复计算|
|`useReducer()`|控制状态更新，减少不必要的 `setState`|
