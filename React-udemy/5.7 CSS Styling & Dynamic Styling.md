# React 项目里的 CSS Styling & Dynamic Styling（详尽讲解 + 可运行示例）

下面把常见的样式方案、它们的优缺点、以及如何做**动态样式**（随 state/props/主题/交互改变样式）系统化地讲清楚，并给出一组小而完整的示例代码。你可以把这些直接拷到项目里运行或改造。

---

## 概览 — React 中常见的样式方案

- **全局 CSS / SCSS**（常规 `.css` / `.scss`）  
    优点：简单、兼容性好；缺点：类名冲突可能，维护大项目时不方便。
    
- **CSS Modules**（`.module.css`）  
    本地作用域，避免冲突，适合组件化项目。
    
- **Inline styles（style 对象）**  
    用于动态值（例如 JS 计算的宽高、颜色）。不能写伪类/媒体查询。
    
- **CSS-in-JS**（styled-components、Emotion 等）  
    强大的 JS 驱动样式，支持 props、主题、SSR（需配置）。
    
- **Utility-first（例如 Tailwind）**  
    快速构建、可组合性高，但类名较多。
    
- **CSS 变量（Custom properties） + 切换 class**  
    非常适合主题切换与运行时动态样式（性能友好）。

# 示例 1 — 全局 CSS & 条件类（基础）

文件结构：

```
src/
  index.css
  App.jsx
```

`index.css`

```css
/* 全局样式 */
.btn {
  padding: 8px 12px;
  border: none;
  border-radius: 6px;
  cursor: pointer;
  transition: background-color 180ms ease;
}
.btn--danger { background-color: #e53e3e; color: white; }
.btn--muted  { background-color: #e2e8f0; color: #1a202c; }
.alert {
  padding: 16px;
  border-radius: 8px;
  background: #fff7ed;
  border: 1px solid #fbd38d;
}
.alert.hidden { display: none; }
```

`App.jsx`

```jsx
import React, { useState } from "react";
import "./index.css";

export default function App() {
  const [showAlert, setShowAlert] = useState(false);

  return (
    <div style={{ padding: 24 }}>
      <div className={`alert ${showAlert ? "" : "hidden"}`} data-testid="alert">
        <h2>Are you sure?</h2>
        <p>These changes can't be reverted!</p>
        <button className="btn btn--muted" onClick={() => setShowAlert(false)}>Proceed</button>
      </div>

      <button
        className="btn btn--danger"
        onClick={() => setShowAlert(true)}
      >
        Delete
      </button>
    </div>
  );
}
```

说明：用字符串拼接或模板字面量切换类名；简单直接。

# 示例 2 — CSS Modules（局部作用域）

文件结构：

```
src/
  Button.module.css
  Button.jsx
```

`Button.module.css`

```css
.button {
  padding: 10px 14px;
  border-radius: 6px;
  border: none;
  cursor: pointer;
  transition: transform 160ms;
}
.primary {
  background: #2563eb;
  color: white;
}
.secondary {
  background: #e5e7eb;
  color: #111827;
}
```

`Button.jsx`

```jsx
import React from "react";
import styles from "./Button.module.css";

export default function Button({ variant = "primary", children, ...rest }) {
  const cls = variant === "primary" ? `${styles.button} ${styles.primary}` : `${styles.button} ${styles.secondary}`;
  return <button className={cls} {...rest}>{children}</button>;
}
```

说明：CSS Modules 会把类名变成局部唯一的（构建期间），避免冲突。

# 示例 3 — styled-components（props 驱动样式 + 主题）

> 需要先安装 `styled-components`：`npm i styled-components`

`App.jsx`

```jsx
import React, { useState } from "react";
import styled, { ThemeProvider } from "styled-components";

const theme = {
  light: { bg: "#ffffff", text: "#111" },
  dark:  { bg: "#0f172a", text: "#e6eef6" }
};

const Card = styled.div`
  background: ${({ theme }) => theme.bg};
  color: ${({ theme }) => theme.text};
  padding: 16px;
  border-radius: 10px;
  box-shadow: 0 6px 18px rgba(2,6,23,0.08);
`;

const Btn = styled.button`
  padding: 8px 12px;
  border-radius: 6px;
  border: none;
  cursor: pointer;
  background: ${({ primary }) => (primary ? "#2563eb" : "#e2e8f0")};
  color: ${({ primary }) => (primary ? "#fff" : "#111")};
  &:hover { transform: translateY(-1px); }
`;

export default function App() {
  const [themeName, setThemeName] = useState("light");
  return (
    <ThemeProvider theme={theme[themeName]}>
      <Card>
        <h3>Theme demo</h3>
        <Btn primary onClick={() => alert("primary")}>Primary</Btn>
        <Btn style={{ marginLeft: 8 }} onClick={() => setThemeName(t => t === "light" ? "dark" : "light")}>
          Toggle Theme
        </Btn>
      </Card>
    </ThemeProvider>
  );
}
```

说明：styled-components 支持把 props、theme 注入样式内，适合组件级复杂样式与主题化。

# 示例 4 — Inline styles（运行时动态数值）与 `useMemo` 优化

```jsx
import React, { useState, useMemo } from "react";

export default function InlineStyleExample() {
  const [size, setSize] = useState(120);
  const style = useMemo(() => ({
    width: size + "px",
    height: size + "px",
    lineHeight: size + "px",
    backgroundColor: size > 140 ? "#ef4444" : "#60a5fa",
    borderRadius: 8,
    textAlign: "center",
    color: "#fff"
  }), [size]);

  return (
    <div>
      <div style={style}>box</div>
      <input type="range" min="50" max="220" value={size} onChange={e => setSize(Number(e.target.value))} />
    </div>
  );
}
```

说明：

- 用 JS 计算动态尺寸或颜色很方便；
    
- **注意**不要在每次 render 都创建新对象（会导致下游 PureComponent / memo 失效），用 `useMemo` 缓存 style 对象。

# 示例 5 — Tailwind / Utility 类（快速构建）

```jsx
export default function TailwindToggle({ active }) {
  return (
    <button className={`px-4 py-2 rounded ${active ? "bg-blue-600 text-white" : "bg-gray-200 text-gray-800"}`}>
      {active ? "On" : "Off"}
    </button>
  );
}
```

说明：Tailwind 高效但依赖构建工具；动态切换只需切换类名字符串。

# 示例 6 — CSS 变量（Custom Properties）用于主题和运行时值（推荐做法）

`index.css`

```css
:root {
  --bg: #ffffff;
  --text: #111;
  --accent: #2563eb;
}
[data-theme="dark"] {
  --bg: #0b1220;
  --text: #e6eef6;
  --accent: #60a5fa;
}
.app {
  background: var(--bg);
  color: var(--text);
  min-height: 100vh;
  padding: 24px;
}
```

`App.jsx`

```jsx
import React, { useState, useEffect } from "react";
import "./index.css";

export default function App() {
  const [theme, setTheme] = useState("light");

  useEffect(() => {
    document.documentElement.setAttribute("data-theme", theme);
  }, [theme]);

  return (
    <div className="app">
      <button onClick={() => setTheme(t => t === "light" ? "dark" : "light")}>
        Toggle theme
      </button>
      <div style={{ marginTop: 12, padding: 12, borderRadius: 8, background: "var(--accent)", color: "white" }}>
        Accent box
      </div>
    </div>
  );
}
```

说明：把主题颜色写成 CSS 变量，切换主题只需切换 `data-theme`，浏览器渲染友好，性能好。

# 动态样式技巧与注意事项（实战经验）

1. **避免频繁创建新 style 对象 / 新函数引用**
    
    - 用 `useMemo`/`useCallback` 缓存，减少无谓 re-render。
        
2. **优先用类切换或 CSS 变量，实现样式切换**（性能好，避免 inline style 覆盖复杂样式）。
    
3. **CSS 动画优先使用 transform/opacity（GPU 加速）**，避免使用 `top/left` 导致回流。
    
4. **Accessibility（可访问性）**：切换类时同步更新 `aria-pressed`、`aria-expanded` 等属性，保证屏读器友好。
    
5. **SSR 注意**：某些 CSS-in-JS 需要 SSR 配置（styled-components/Emotion）以防闪烁（FOUC）。
    
6. **测试**：用 `data-testid` 或语义选择器测试样式相关行为（比如是否存在 `class` 或 `aria` 属性）。
    
7. **样式优先级**：了解 CSS specificity；避免滥用 `!important`。
    
8. **主题切换保持用户偏好**：用 `localStorage` 或 `prefers-color-scheme` 初始值。

# 常见问题答疑（FAQ）

- **什么时候用 inline style？**  
    运行时需要直接注入数值（宽高、颜色、transform 值），或动态计算样式时用；但复杂样式用 class 或 CSS-in-JS 更好。
    
- **样式冲突如何避免？**  
    使用 CSS Modules、BEM 或 CSS-in-JS（自动命名）避免全局冲突。
    
- **性能问题？**  
    动态样式尽量使用 CSS 类或 CSS 变量切换，避免为动画或高频更新使用 inline style 的属性会导致重排（reflow）。
    
- **如何做主题切换？**  
    推荐用 CSS 变量 + 在 `html`/`body` 或最外层元素切换 `data-theme` 类/属性；也可以用 styled-components 的 `ThemeProvider`。

## 结尾（快速建议）

- 小项目或传统团队：用**全局 CSS / SCSS + BEM** 或 **CSS Modules**。
    
- 组件库或需要强交互/主题：**styled-components / Emotion + ThemeProvider**。
    
- 需要快速开发且能接受工具链：**Tailwind**。
    
- 运行时大量改变颜色/间距/主题：**CSS 变量 + class 切换**（首选）。
